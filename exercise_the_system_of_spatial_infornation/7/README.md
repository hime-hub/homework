## 概要

Google Colab（クラウド GPU）上で `tf.keras` を用いて道路クラック検出モデルを学習します。学習は 50 エポックを想定しており、約 2 分で完了します。

## 学習目標

- 3 層の畳み込みネットワーク（Dropout・Batch Normalization 付き）を Keras で実装する

- モデルの出力結果を解釈する

**タスク定義**

```
入力: (256, 256, 3) の道路画像 → F(x) → 出力: 0（クラックあり）/ 1（損傷なし）
```

## モデル

### 1. Simple Neural Network

```
Input(256×256) → Flatten → Dense(512, relu) → Dropout(0.2)
               → Dense(512, relu) → Dropout(0.2) → Dense(2, softmax)
```

### 2. CNN（カスタム 3 層）

```
BatchNorm → Conv2D(64, 7×7) → MaxPool → Dropout(0.4)
          → BatchNorm → Conv2D(128, 3×3) → MaxPool → Dropout(0.4)
          → BatchNorm → Conv2D(256, 5×5) → MaxPool → Dropout(0.4)
          → Flatten → Dense(256, elu) → Dropout(0.5) → Dense(2, softmax)
```

### 3. ResNet50（転移学習）

ImageNet 事前学習済みの ResNet50 にカスタム分類ヘッドを追加。

## 宿題

### 問題 1（20 点）

あなたは地方自治体の道路管理者です。あるシステムエンジニアリング会社が「精度 99% の道路点検ソフトウェアを開発しました！ぜひ購入してください！」と言ってきました。あなたは何を疑うべきでしょうか？

**回答：クラス不均衡による見かけ上の高精度を疑う。**

道路データセットは損傷箇所より正常箇所の方が圧倒的に多い。今回のデータでも損傷 167 枚・正常 224 枚とすでに偏りがある。現実の道路では損傷はさらに少ない。

仮に損傷が 1%・正常が 99% の場合、「常に正常と予測するだけ」のモデルでも精度 99% が出る。Confusion Matrix の結果を見ると、実際に損傷クラスの Recall が 0 になっているケースが確認できた（実習の ResNet50 結果がまさにそれ）。

**確認すべき指標：**

| 指標 | 理由 |
|---|---|
| Recall（再現率） | 「本物の損傷を見逃していないか」を測る最重要指標 |
| Precision（適合率） | 「損傷と判定した中に誤報がどれだけあるか」 |
| F1 スコア | Precision と Recall の調和平均。不均衡データで最も信頼できる総合指標 |
| 混同行列 | FN（見逃し）の件数を直接確認する |

道路管理の文脈では **FN（見逃し）のコストが非常に高い** ため、精度よりも Recall を優先して評価すべきである。

### 問題 2（20 点）

コンピューターは疲れることがなく、いつまでもモデルを学習し続けることができます。しかし、エポック数を際限なく増やすことは現実的ではありません。その理由を考え、説明してください。

**回答：過学習（Overfitting）が発生するから。**

エポックを重ねるほどモデルは訓練データの細かいノイズや外れ値まで「暗記」するようになる。その結果、訓練損失は下がり続けるが、未知のデータ（検証・テスト）に対する損失は逆に上昇し始める。

```
         訓練損失 ↘ （ずっと下がる）
検証損失  ↘ → ↗  ← ここが「過学習の始まり」
                ↑
          最適な停止点
```

**他の理由：**
- 学習時間・計算コストが線形に増加する
- 学習率が固定の場合、損失がある値で振動し続け改善しなくなる（収束済み）

**対策として用いる手法：**
- Early Stopping（検証損失が改善しなくなったら自動停止）
- ReduceLROnPlateau（損失が頭打ちになったら学習率を下げる）
- Dropout・Batch Normalization（過学習の抑制）

### 問題 3（30 点）

学習率やその他のハイパーパラメータ、あるいはニューラルネットワークの構造を少し変更してみてください。そして、その結果がサンプルとどのように異なるかを確認してください。（ネットワークの設定と学習結果のスクリーンショットを添付しても構いません。）

**回答：代表的な変更例とその効果を示す。**

**学習率を変えた場合**

| 学習率 | 挙動 |
|---|---|
| 大きすぎる (0.1) | 損失が振動・発散し学習が不安定 |
| 適切 (0.0001) | 安定して収束 |
| 小さすぎる (0.000001) | 収束が非常に遅く、エポック数が不足する |

**Dropout 率を変えた場合**

| Dropout 率 | 挙動 |
|---|---|
| 0（なし） | 訓練精度は高いが検証精度が低い（過学習） |
| 0.4〜0.5 | 汎化性能が向上（サンプルの設定） |
| 0.8（過剰） | モデルが学習できず訓練・検証ともに精度が低い（未学習） |

**エポック数を増やした場合（5 → 20）**

- 訓練データが少ない場合は過学習リスクが高まる
- Dropout・BN が入っているため緩和はされるが、検証損失の推移を確認することが必要

### 問題 4（30 点）

入力画像のサイズを 256×256 から 512×512 に変更した場合、各層のパラメータ数はどのように変わりますか？また、すべての畳み込みカーネルのサイズを 5×5 から 3×3 に変更した場合、各層のパラメータ数はどのように変わりますか？計算の詳細を示してください。（Colab の講義資料にある CNN モデルをもとに回答しても構いません。）

**回答：カスタム CNN（3 層）を基準に計算する。条件は入力 256×256、グレースケール（1 チャンネル）、padding='same'。**

Conv2D のパラメータ数の公式：

```
(カーネル縦 × カーネル横 × 入力チャンネル数 + 1) × フィルタ数
```

**元のパラメータ数**

| 層 | 計算式 | パラメータ数 |
|---|---|---|
| Conv2D(64, 7×7) | (7×7×1 + 1) × 64 | 3,200 |
| Conv2D(128, 3×3) | (3×3×64 + 1) × 128 | 73,856 |
| Conv2D(256, 5×5) | (5×5×128 + 1) × 256 | 819,456 |
| Flatten → Dense(256) | (32×32×256 + 1) × 256 | 67,109,120 |
| Dense(2) | (256 + 1) × 2 | 514 |

Flatten のサイズ計算（padding=same のため Conv2D は空間サイズを変えない）：

```
256×256 → MaxPool(2,2) → 128×128
         → MaxPool(2,2) →  64×64
         → MaxPool(2,2) →  32×32×256
Flatten → 32 × 32 × 256 = 262,144
```

**4a：入力を 256×256 → 512×512 に変えた場合**

Conv2D のパラメータ数は「カーネルサイズ × チャンネル数」にのみ依存し、入力の空間サイズには依存しないため Conv2D 層は変化なし。一方 Flatten 後のサイズが 4 倍になるため Dense(256) のパラメータ数も 4 倍になる。

```
512×512 → MaxPool → 256×256
         → MaxPool → 128×128
         → MaxPool →  64×64×256
Flatten → 64 × 64 × 256 = 1,048,576（262,144 の 4 倍）
```

| 層 | 256×256 入力 | 512×512 入力 | 変化 |
|---|---|---|---|
| Conv2D(64, 7×7) | 3,200 | 3,200 | 変化なし |
| Conv2D(128, 3×3) | 73,856 | 73,856 | 変化なし |
| Conv2D(256, 5×5) | 819,456 | 819,456 | 変化なし |
| Dense(256) | 67,109,120 | 268,435,712 | **約 4 倍** |
| Dense(2) | 514 | 514 | 変化なし |

**4b：5×5 カーネルを 3×3 に変えた場合**

第 3 Conv2D(256, 5×5) のみ変化する。padding='same' のため空間サイズは変わらず、Flatten 以降の Dense 層には影響しない。

```
変更前: (5×5×128 + 1) × 256 = 819,456
変更後: (3×3×128 + 1) × 256 = 295,168
削減量: 819,456 - 295,168 = 524,288 パラメータ減
```

| 層 | 5×5 カーネル | 3×3 カーネル | 変化 |
|---|---|---|---|
| Conv2D(256, 5×5 → 3×3) | 819,456 | 295,168 | **約 36% に減少** |
| Dense(256) | 67,109,120 | 67,109,120 | 変化なし |

**まとめ：** 入力サイズの変更は Conv2D に影響せず Dense に影響する。カーネルサイズの変更は Conv2D に影響し Dense には影響しない。
