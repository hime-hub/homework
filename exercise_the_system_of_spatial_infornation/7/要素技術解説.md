## 1. モデルアーキテクチャ

### 1.1 シンプルニューラルネットワーク（全結合ネットワーク）

全結合層（Dense 層）だけで構成された最もシンプルな分類モデル。

```
Input(256×256) → Flatten → Dense(512, ReLU) → Dropout(0.2)
               → Dense(512, ReLU) → Dropout(0.2) → Dense(2, Softmax)
```

**特徴**
- 構造がシンプルで実装・デバッグが容易。
- 画像を一次元に展開（Flatten）して処理するため、**空間的な特徴（形状・テクスチャ）を考慮しない**。
- パラメータ数が非常に多くなりやすく、過学習しやすい。
- 本講義では比較ベースラインとして使用。エポック数は 200 に設定。

---

### 1.2 CNN（畳み込みニューラルネットワーク）

Convolutional Neural Network。Conv2D・MaxPooling・BN・Dropout を組み合わせたカスタム 3 層モデル。

```
BN → Conv2D(64, 7×7) → MaxPool → Dropout(0.4)
   → BN → Conv2D(128, 3×3) → MaxPool → Dropout(0.4)
   → BN → Conv2D(256, 5×5) → MaxPool → Dropout(0.4)
   → Flatten → Dense(256, ELU) → Dropout(0.5) → Dense(2, Softmax)
```

**特徴**
- カーネル（フィルタ）が画像上をスライドして局所的な特徴（エッジ、テクスチャ）を抽出する。
- 同じフィルタを画像全体に共有（重み共有）するため、全結合層より大幅にパラメータが少ない。
- 全結合モデルと異なり、画像の**平行移動不変性**を持つ。
- 本講義では入力サイズ 256×256、RGB（3 チャンネル）を想定。

---

### 1.3 ResNet50・転移学習

**転移学習（Transfer Learning）** とは、大規模データセットで事前学習されたモデルの重みを再利用して、少量データの新しいタスクを解く手法。

**ResNet50** は深さ 50 層の残差ネットワーク（Residual Network）で、ImageNet（120 万枚超）で事前学習済み。

```
ResNet50（ImageNet 事前学習済み・重み固定）
    ↓
カスタム分類ヘッド（全結合 + Softmax）
```

**残差接続（Skip Connection）**：

```
出力 = F(x) + x
```

層の出力に入力 x を加算することで勾配消失を防ぎ、非常に深いネットワークでも安定した学習を実現する。

**転移学習の利点**
- データが少なくても高精度が期待できる。
- 学習時間が短い（特徴抽出部分は固定するため）。

**注意点**（本講義での観察）
- クラス不均衡データに対して「常に多数クラスを予測」するモデルに収束するケースがあった（Recall = 0）。事前学習の恩恵を活かすには、クラス不均衡への対策が別途必要。

## 2. レイヤー・構成要素

### 2.1 Conv2D（畳み込み層）

2 次元畳み込み演算を行う層。

**パラメータ数の公式**：

```
(カーネル縦 × カーネル横 × 入力チャンネル数 + 1) × フィルタ数
```

`+1` はバイアス項。

**例**（入力チャンネル 1、グレースケール）：

| 層 | 計算 | パラメータ数 |
|---|---|---|
| Conv2D(64, 7×7) | (7×7×1 + 1) × 64 | 3,200 |
| Conv2D(128, 3×3) | (3×3×64 + 1) × 128 | 73,856 |
| Conv2D(256, 5×5) | (5×5×128 + 1) × 256 | 819,456 |

**padding='same'** を指定すると、出力の空間サイズが入力と同じになる（ゼロパディングで補填）。

---

### 2.2 MaxPooling

特徴マップを縮小し、**最も顕著な特徴だけを残す**プーリング層。

`MaxPool(2×2)` の場合、2×2 領域の最大値を取り出し、空間サイズを 1/2 に縮小する。

```
128×128 → MaxPool(2,2) → 64×64
```

**効果**
- パラメータ数の削減（計算コスト低減）。
- 微小なズレ・変形に対するロバスト性の向上。

---

### 2.3 Flatten

多次元テンソルを 1 次元ベクトルに変換する層。Conv2D や MaxPooling の出力を Dense 層に接続するために使用する。

```
32×32×256 → Flatten → 262,144
```

Flatten 後の次元数は続く Dense 層のパラメータ数に直接影響する。入力画像サイズを大きくすると Dense 層のパラメータ数が増大する原因がここにある。

---

### 2.4 Dense（全結合層）

すべての入力ニューロンと出力ニューロンが結合された層。

**パラメータ数の公式**：

```
(入力次元 + 1) × 出力次元
```

例：`Dense(256)` で入力 32×32×256=262,144 の場合：

```
(262,144 + 1) × 256 = 67,109,120
```

出力層には通常 Softmax を使用し、クラス確率を出力する。

---

## 3. 活性化関数

### 3.1 ReLU（Rectified Linear Unit）

```
f(x) = max(0, x)
```

**特徴**
- 最も広く使われる活性化関数。
- x > 0 の範囲では線形なので勾配消失が起きにくい。
- x ≤ 0 の出力が常に 0 になる「死んだニューロン（Dying ReLU）」問題がある。
- 本モデルでは Simple NN の隠れ層に使用。

---

### 3.2 ELU（Exponential Linear Unit）

```
f(x) = x          (x > 0)
f(x) = α(e^x - 1) (x ≤ 0)  ※ α は通常 1.0
```

**特徴**
- x ≤ 0 でも滑らかな負の出力を持つため、Dying ReLU 問題が軽減される。
- ReLU より収束が速くなる場合がある。
- 本モデルでは CNN の全結合層（`Dense(256, elu)`）に使用。

---

### 3.3 Softmax

```
softmax(z_i) = e^{z_i} / Σ_j e^{z_j}
```

**特徴**
- 出力ベクトルの各要素を [0, 1] に変換し、合計を 1 にする（確率分布）。
- 多クラス分類の出力層に使用。
- 本モデルでは `Dense(2, softmax)` で「クラックあり / 損傷なし」の 2 クラス確率を出力。

---

## 4. 正則化・学習安定化

### 4.1 Dropout

学習中にランダムに指定した割合のニューロンを「無効化」することで、特定ニューロンへの依存を防ぐ正則化手法。

**挙動**
- 学習時：指定した確率 p でニューロンを無効化する。
- 推論時：すべてのニューロンを使用し、出力を (1-p) でスケーリング（または学習時にスケーリング）。

| Dropout 率 | 挙動 |
|---|---|
| 0（なし） | 過学習しやすい |
| 0.4〜0.5 | 汎化性能が向上（本モデルの設定） |
| 0.8（過剰） | 学習不足（未学習）に陥る |

---

### 4.2 Batch Normalization（BN）

ミニバッチ内の活性化値を正規化（平均 0、分散 1 に近づける）し、学習を安定させる手法。

**効果**
- 内部共変量シフト（Internal Covariate Shift）を軽減し、学習率を大きくできる。
- 収束速度が向上する。
- 正則化効果もあり、Dropout と組み合わせることが多い。

本モデルでは各 Conv2D の前に BN を配置している（`BatchNorm → Conv2D`）。

---

### 4.3 Early Stopping

検証損失（Validation Loss）が改善しなくなった時点で学習を自動停止するコールバック。

```
訓練損失 ↘（ずっと下がる）
検証損失 ↘ → ↗  ← 過学習の始まり ＝ Early Stopping の停止点
```

**設定例（Keras）**：

```python
EarlyStopping(monitor='val_loss', patience=5, restore_best_weights=True)
```

`patience` は「何エポック改善がなければ停止するか」を指定する。

---

### 4.4 ReduceLROnPlateau

検証損失が一定エポック以上改善しない場合に、学習率を自動的に下げるコールバック。

**設定例（Keras）**：

```python
ReduceLROnPlateau(monitor='val_loss', factor=0.5, patience=3)
```

学習が進んで損失の改善が鈍化したとき、学習率を小さくすることで最小値付近をより精密に探索する。

---

## 5. 学習・最適化

### 5.1 AdamW オプティマイザ

Adam（Adaptive Moment Estimation）に **Weight Decay（重み減衰）** を追加した最適化アルゴリズム。

**Adam の特徴**
- 各パラメータに対して適応的な学習率を計算する。
- 勾配の一次モーメント（平均）と二次モーメント（分散）の推定値を使用。

**Weight Decay の追加（W = AdamW）**
- L2 正則化と似ているが、重み減衰を勾配ではなく直接パラメータに適用する。
- 過学習の抑制効果がある。

本講義での設定：`lr=0.0001`

---

### 5.2 過学習（Overfitting）

訓練データに「最適化されすぎ」て、未知のデータへの汎化性能が低下する現象。

```
エポックが増えるにつれ：
  訓練損失  → 下がり続ける
  検証損失  → 下がった後、上昇に転じる  ← 過学習
```

**原因**
- モデルが訓練データのノイズや外れ値まで記憶してしまう。

**対策**
- Dropout、Batch Normalization による正則化。
- Early Stopping による学習の早期停止。
- データ拡張によるデータ量の仮想的な増加。
- ReduceLROnPlateau による学習率の調整。

---

### 5.3 ハイパーパラメータ（学習率・バッチサイズ・エポック）

モデルが学習によって決定するパラメータとは異なり、**人間が事前に設定する設計値**。

| パラメータ | 本講義の設定 | 影響 |
|---|---|---|
| 学習率 (lr) | 0.0001 | 大きすぎると発散、小さすぎると収束が遅い |
| バッチサイズ | 8 | 小さいほどノイズが多く汎化しやすいが学習が不安定になりやすい |
| エポック数 | CNN/ResNet: 5、Simple NN: 200 | 多すぎると過学習、少なすぎると未学習 |

**学習率の影響**：

| 学習率 | 挙動 |
|---|---|
| 大きすぎる (0.1) | 損失が振動・発散 |
| 適切 (0.0001) | 安定して収束 |
| 小さすぎる (0.000001) | 収束が非常に遅い |

---

### 5.4 データ拡張（Data Augmentation）

既存の画像に変換を加えて仮想的にデータを増やす手法。過学習抑制と汎化性能向上が目的。

本講義で使用した変換：

| 手法 | 内容 |
|---|---|
| 水平反転 | 左右に鏡像反転 |
| 垂直反転 | 上下に鏡像反転 |
| シフト | 画像を縦横方向にランダムにずらす |
| ズーム | ランダムに拡大・縮小 |
| 回転 | ランダムに角度を変えて回転 |

## 6. 評価指標・エラー分析

### 6.1 混同行列（Confusion Matrix）と TP/TN/FP/FN

予測結果を実際のラベルと比較した 2×2 の表。

|  | 予測：Positive（クラックあり） | 予測：Negative（損傷なし） |
|---|---|---|
| **実際：Positive** | TP（真陽性：正しく検出） | FN（偽陰性：見逃し） |
| **実際：Negative** | FP（偽陽性：誤検知） | TN（真陰性：正しく正常） |

**道路管理の文脈**では FN（損傷の見逃し）のコストが特に高い。本講義の Error Analysis セクションでは TP/TN/FP/FN の画像を可視化して誤分類のパターンを確認する。

---

### 6.2 Precision（適合率）

```
Precision = TP / (TP + FP)
```

「損傷と予測した中で、実際に損傷だった割合」。誤検知（FP）が多いと下がる。

---

### 6.3 Recall（再現率）

```
Recall = TP / (TP + FN)
```

「実際の損傷のうち、正しく検出できた割合」。見逃し（FN）が多いと下がる。

道路管理など **FN のコストが高いタスク**では、Recall が最重要指標となる。

---

### 6.4 F1 スコア

Precision と Recall の調和平均。

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

**特徴**
- どちらか一方が極端に低い場合、F1 も低い値になる。
- クラス不均衡データでは Accuracy より信頼できる総合指標。

---

### 6.5 クラス不均衡（Class Imbalance）

データセット内のクラス間でサンプル数に大きな偏りがある状態。

**本講義での例**

| クラス | サンプル数 |
|---|---|
| 000_damage（クラックあり） | 167 枚 |
| 001_no_damage（損傷なし） | 224 枚 |

現実の道路では損傷箇所はさらに少ない（例：損傷 1%・正常 99%）。

**問題点**：「常に正常と予測する」だけのモデルでも Accuracy 99% を達成できる。このため、Accuracy だけで評価すると**モデルが実際には使い物にならないにもかかわらず高スコアに見える**。

**対策**：Recall・Precision・F1 スコアや混同行列を使って評価する。

## 7. データ処理

### 7.1 データ準備（Data Preparation）

Google Drive 上の `lecture_data.tar` からデータセットをダウンロードし、展開・分割する工程。

- 学習・検証・テストセットへの分割は scikit-learn の `train_test_split` を使用。
- 画像は 256×256 px（ResNet50 使用時は 224×224 px）にリサイズ。
- ピクセル値を [0, 1] に正規化（`/ 255.0`）。

---

### 7.2 画像操作（NumPy による配列操作・反転・クロップ）

画像を NumPy の多次元配列として扱い、Python コードで操作する手法。

| 操作 | NumPy 操作例 | 説明 |
|---|---|---|
| 水平反転 | `np.fliplr(img)` | 左右を反転 |
| 垂直反転 | `np.flipud(img)` | 上下を反転 |
| ランダムクロップ | 配列スライス `img[y:y+h, x:x+w]` | 指定領域を切り抜く |

画像は shape `(256, 256, 3)` の 3 次元配列（高さ × 幅 × チャンネル）で表現される。

---

### 7.3 ImageNet（事前学習データセット）

120 万枚超の画像と 1,000 クラスのラベルを持つ大規模画像データセット。深層学習モデルの事前学習に広く使用される。

ResNet50 は ImageNet で事前学習された重みを初期値として持つため、エッジ・テクスチャ・形状など汎用的な特徴を最初から認識できる。これがデータ量が少ない新タスクへの転移学習で有効な理由。

## 8. ライブラリ・実行環境

### 8.1 TensorFlow / Keras (tf.keras)

Google が開発したオープンソースの機械学習フレームワーク。Keras は TensorFlow の高レベル API で、モデルの構築・学習・評価を簡潔に記述できる。

```python
from tensorflow import keras
model = keras.Sequential([...])
model.compile(optimizer='adamw', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(X_train, y_train, epochs=5, validation_data=(X_val, y_val))
```

GPU を利用した高速計算が可能で、本講義では Google Colab の GPU 環境で動作させる。

---

### 8.2 NumPy

Python の数値計算ライブラリ。多次元配列（ndarray）を効率的に扱う。

画像処理での主な用途：
- 画像を ndarray として読み込み・操作する。
- 反転・クロップなどの前処理。
- ピクセル値の正規化（`img / 255.0`）。

---

### 8.3 scikit-learn

Python の機械学習ユーティリティライブラリ。モデル構築よりも**データ分割・評価**に使用。

本講義での使用目的：
- `train_test_split`：データセットを訓練・検証・テストに分割。
- `confusion_matrix`：混同行列の計算。
- `classification_report`：Precision / Recall / F1 スコアの一括出力。

---

### 8.4 Matplotlib / PIL

**Matplotlib**：Python のデータ可視化ライブラリ。学習曲線（損失・精度の推移）や画像の表示に使用。

**PIL（Pillow）**：Python 画像処理ライブラリ。画像ファイルの読み込み・リサイズ・フォーマット変換などに使用。

---

### 8.5 Google Colab

Google が提供するクラウド上の Jupyter Notebook 実行環境。

**特徴**
- GPU（Tesla T4 等）を無料で利用できる。
- セットアップ不要で Python/TensorFlow 環境がすぐに使える。
- Google Drive とのシームレスな連携が可能。

本講義では GPU を使って CNN 学習を約 2 分で完了させる。

---

### 8.6 PyDrive / google-api-python-client

Google Drive へのアクセスを Python から行うためのライブラリ。

- **PyDrive**：Google Drive API をラップしたシンプルなインターフェース。ファイルのアップロード・ダウンロードを数行で実現。
- **google-api-python-client**：Google の各種 API（Drive, Sheets 等）の公式 Python クライアント。
