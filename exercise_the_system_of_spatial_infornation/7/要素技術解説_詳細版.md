# 第0章 前提知識：機械学習とニューラルネットワークの基礎

## 0.1 機械学習とは何か

### 従来のプログラミングとの違い

従来のコンピュータプログラムは、「もし〇〇ならば××する」というルールを人間が明示的に書いていた。たとえば「クラックのある道路画像を判定するプログラム」を作ろうとしたら、「ピクセルの輝度差がある線が一定以上あれば損傷と判定する」といったルールを人間が考えて書く必要があった。

しかし現実の道路損傷は形が多様で、照明条件も天気も違い、ルールを手で書くのは事実上不可能だ。

**機械学習**はこのアプローチを逆転させる。

```
従来：プログラム（ルール） + データ → 答え
機械学習：データ + 答え（正解ラベル） → プログラム（ルール）を自動で学習
```

つまり「クラックのある画像」と「ない画像」をたくさん見せることで、コンピュータが自動的に判別のルール（モデル）を獲得する。

### 教師あり学習

本講義は**教師あり学習**の一種。正解ラベル付きのデータを使ってモデルを学習する。

- 入力 x：道路画像（256×256 ピクセル、RGB）
- 出力 y：0（クラックあり）または 1（損傷なし）

モデルは「この画像はクラックありの確率が 87% です」のような形で予測を出力し、正解と比較しながら少しずつ改善していく。

---

## 0.2 ニューラルネットワークの基本構造

### ニューロンとは

ニューラルネットワーク（Neural Network）は、脳の神経細胞（ニューロン）のネットワーク構造を数学的にモデル化したもの。

1 つのニューロン（人工ニューロン）は以下の計算をする：

```
出力 z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
出力 a = activation(z)
```

- `x₁, x₂, ..., xₙ`：入力値（前の層からの信号）
- `w₁, w₂, ..., wₙ`：**重み（Weight）**。各入力の「重要度」を表す。
- `b`：**バイアス（Bias）**。出力全体をシフトするための定数項。
- `activation`：**活性化関数**（後述）。線形変換に非線形性を加える。

**具体例：** 2 つの入力 x₁=0.5, x₂=0.3 に対して、重み w₁=2.0, w₂=-1.0, バイアス b=0.1 のニューロンは：

```
z = 2.0×0.5 + (-1.0)×0.3 + 0.1 = 1.0 - 0.3 + 0.1 = 0.8
a = ReLU(0.8) = 0.8
```

### 層（Layer）の構造

ニューロンを並べてまとめたものを**層（Layer）**と呼ぶ。

```
[入力層] → [隠れ層1] → [隠れ層2] → ... → [出力層]
```

- **入力層**：データをそのまま受け取る（256×256×3 = 196,608 個のニューロン）
- **隠れ層**：特徴を抽出・変換する（何層でも積み重ねられる）
- **出力層**：最終的な予測を出す（2 クラス分類なら 2 個のニューロン）

「深層（Deep）」学習とは、隠れ層が多い（深い）ニューラルネットワークのこと。

### 重みとバイアスはどう決まるか

最初は**ランダムに初期化**される。学習を通じて「正解に近づくように」更新されていく（後述の勾配降下法）。

**重みとバイアスの総数 = パラメータ数**と呼ぶ。本講義の CNN は数千万パラメータを持つ。

---

## 0.3 順伝播（Forward Propagation）

入力データをネットワークに流して予測を得るプロセスを**順伝播**と呼ぶ。

```
画像 x  
  ↓（Conv2D で特徴抽出）  
特徴マップ  
  ↓（MaxPooling で縮小）  
  ↓（Flatten で 1 次元化）  
特徴ベクトル  
  ↓（Dense で変換）  
[クラックあり確率 0.87, 損傷なし確率 0.13]
```

順伝播は「予測を出すだけ」で、重みはまだ更新しない。

---

## 0.4 損失関数（Loss Function）

**損失関数**（またはコスト関数）は、モデルの予測がどれだけ間違っているかを数値（損失値）で表す関数。この値が小さいほど「良い予測」である。

### 交差エントロピー損失（Categorical Crossentropy）

分類問題でよく使われる損失関数。

```
L = -Σ y_true × log(y_pred)
```

- `y_true`：正解ラベル（例：クラックあり → [1, 0]）
- `y_pred`：モデルの予測確率（例：[0.87, 0.13]）

**直感的な理解：**  
正解クラスの予測確率が高ければ損失は小さく（良い）、低ければ損失は大きい（悪い）。

```
正解がクラック「あり」で、予測確率 0.87 の場合：
L = -log(0.87) ≈ 0.14  （小さい → 良い）

正解がクラック「あり」で、予測確率 0.01 の場合：
L = -log(0.01) ≈ 4.61  （大きい → 悪い）
```

学習の目標は「損失関数の値を最小化すること」。

---

## 0.5 勾配降下法（Gradient Descent）

損失関数を最小化するために重みを更新する方法。

### 直感的なイメージ

山の斜面を転がり落ちるボールをイメージする。損失関数の値が「高度」で、重みの値が「位置」に対応する。ボールは常に「最も急な下り坂の方向」に転がっていく。この方向が**勾配の逆方向（負の勾配方向）**。

```
新しい重み = 古い重み - 学習率 × 勾配
w ← w - lr × ∂L/∂w
```

- `lr`（学習率）：一度の更新でどれだけ移動するか。大きすぎると飛び越えて発散、小さすぎると収束が遅い。
- `∂L/∂w`：損失 L の重み w に対する偏微分（勾配）。損失を最も大きく増加させる方向を示す。

### ミニバッチ勾配降下法

全データで勾配を計算するのは計算量が膨大なので、**バッチサイズ**分のデータで勾配を計算し、更新する方法（ミニバッチ勾配降下法）が一般的。本講義ではバッチサイズ = 8。

---

## 0.6 誤差逆伝播法（Backpropagation）

勾配降下法では `∂L/∂w`（損失の重みに対する偏微分）が必要。これを効率よく計算するアルゴリズムが**誤差逆伝播法（Backpropagation）**。

### 仕組み

微積分の**連鎖律（Chain Rule）**を使い、出力層から入力層に向かって逆向きに勾配を計算していく。

```
出力層の誤差 → 最終層の重みへの勾配を計算
            ↓
       一つ前の層へ誤差を「逆伝播」
            ↓
       その層の重みへの勾配を計算
            ↓
       （繰り返し）
            ↓
       すべての層の重みの勾配が求まる
```

TensorFlow/Keras では `model.fit()` を呼ぶだけで自動的に順伝播 → 損失計算 → 逆伝播 → 重み更新 が繰り返される。

### 勾配消失問題（Vanishing Gradient Problem）

ネットワークが深くなると、逆伝播中に勾配が何度も小さな値（< 1）を掛け合わされて急激に小さくなり、入力層付近の重みがほとんど更新されなくなる問題。

```
層が 50 層あって、各層で勾配が 0.5 倍になると：
0.5^50 ≈ 0.0000000000000009 → ほぼゼロ
```

これが ResNet の残差接続（Skip Connection）が解決した核心的な問題（1.3 節で詳述）。

---

## 0.7 訓練・検証・テスト分割

手持ちのデータを 3 つに分割して使う。

| 分割 | 役割 | 注意点 |
|---|---|---|
| **訓練セット（Train）** | モデルの重みを学習するために使う | 最も多い割合（例：70〜80%）|
| **検証セット（Validation）** | 学習中にモデルの性能を確認する。過学習の検出に使う | 学習には使わない |
| **テストセット（Test）** | 学習完了後、最終的な性能評価に一度だけ使う | 学習にも検証にも使わない |

**なぜ分ける必要があるのか：**  
訓練データで評価すると「暗記」の度合いしか測れず、未知データへの対応力（汎化性能）を正しく評価できない。検証セットで過学習を監視し、テストセットで「本番さながら」の評価を行う。

# 第1章 モデルアーキテクチャ

## 1.1 シンプルニューラルネットワーク（全結合ネットワーク）

### 構造

```
Input(256×256×3 = 196,608) → Flatten → Dense(512, ReLU) → Dropout(0.2)
                           → Dense(512, ReLU) → Dropout(0.2) → Dense(2, Softmax)
```

本講義でのエポック数：200

### 動作の流れ

1. 256×256 の RGB 画像（3 チャンネル）を受け取る。
2. Flatten で 196,608 個の数値の 1 次元ベクトルに変換する。
3. Dense(512) に渡す。各ニューロンが 196,608 個の入力すべてと接続する（「全結合」の由来）。
4. Dropout で一部ニューロンをランダムに無効化。
5. もう一度 Dense(512) と Dropout を通す。
6. 最後の Dense(2) で「クラックあり」「損傷なし」の 2 クラスの確率を出力する。

### なぜシンプル NN では画像処理に不向きなのか

全結合層は入力の空間的な配置を無視する。ピクセルを単なる数値のリストとして扱うため、「この位置のピクセルと隣のピクセルが連続した線を形成している」という情報が活かせない。

また、196,608 個の入力 × 512 個のニューロン = 約 1 億個のパラメータが第 1 層だけで発生し、計算量と過学習リスクが非常に大きい。

## 1.2 CNN（畳み込みニューラルネットワーク）

### 構造

```
BN → Conv2D(64, 7×7) → MaxPool → Dropout(0.4)
   → BN → Conv2D(128, 3×3) → MaxPool → Dropout(0.4)
   → BN → Conv2D(256, 5×5) → MaxPool → Dropout(0.4)
   → Flatten → Dense(256, ELU) → Dropout(0.5) → Dense(2, Softmax)
```

エポック数：5

### CNN の核心：局所的特徴抽出と重み共有

CNN が全結合 NN と根本的に異なる点は **「小さなフィルタ（カーネル）を画像全体にスライドさせる」** という仕組みにある。

- 全結合層：入力全体と出力全体が結合 → パラメータ数が爆発
- 畳み込み層：小さなフィルタ（例：7×7）だけを学習し、それを画像全体で共有 → パラメータ数を大幅に削減

この「画像全体でフィルタを共有する」仕組みを**重み共有（Weight Sharing）**と呼ぶ。

### 階層的な特徴抽出

CNN の浅い層（最初の畳み込み層）は単純な特徴（エッジ、点）を検出し、深い層に行くほど複雑な特徴（テクスチャ、形状）を検出するようになる。

```
第 1 層（Conv2D 64フィルタ）：エッジや輝度の変化を検出
第 2 層（Conv2D 128フィルタ）：エッジの組み合わせ（角、曲線）を検出
第 3 層（Conv2D 256フィルタ）：より複雑なパターン（クラック状の構造）を検出
```

この階層的な特徴抽出こそが、CNN が画像処理で優れた性能を発揮する理由。

## 1.3 ResNet50・転移学習

### 転移学習とは

まったく新しいタスクをゼロから学習するのではなく、別の大規模タスクで学習済みのモデルを再利用する手法。

**人間の例で考える：** 自動車の運転を覚えた人がトラックを運転する際、「ハンドルを動かすと向きが変わる」「ブレーキで止まる」という知識はゼロから学び直さなくても使える。CNN も同様に、ImageNet で学習したエッジ検出・テクスチャ認識の能力を道路損傷検出に転用できる。

### ResNet50 の構造

ResNet（Residual Network）は 2015 年に Microsoft Research が発表した深層 CNN。50 層の畳み込み層を持つ。

従来の深いネットワークは**勾配消失問題**（0.6 節参照）で深くなるほど学習が困難になっていた。ResNet はこれを**残差接続（Skip Connection）**で解決した。

### 残差接続（Skip Connection）の仕組み

通常の層：
```
出力 H(x) = F(x)
```

残差接続を使った層：
```
出力 H(x) = F(x) + x
```

入力 `x` を変換後の出力 `F(x)` に直接加算する。この「ショートカット」があることで：

1. 勾配が逆伝播する際に、乗算を経ずに直接浅い層まで届く経路が生まれる。
2. 各層は「ゼロ変換 F(x)=0」から始めれば恒等写像 H(x)=x になる。つまり「必要な変換だけを学習すれば良い」ので学習が容易になる。

これにより 50 層どころか 150 層以上でも安定した学習が可能になった。

### 転移学習の具体的な手順

```
① ImageNet 学習済み ResNet50 の重みを読み込む（特徴抽出部分）
② 最後の分類層だけを取り除く
③ 新しい分類ヘッドを追加する（Dense → Softmax で 2 クラス出力）
④ 特徴抽出部分の重みは固定（freeze）して、分類ヘッドだけ学習する
```

本講義では入力サイズを 224×224 に変更している（ResNet50 の学習時のサイズに合わせるため）。

### ImageNet 学習済みモデルが強い理由

ImageNet は犬・猫・車・植物など 1,000 クラス・120 万枚の多様な画像データ。この膨大なデータで学習された ResNet50 は、画像の「汎用的な特徴」を抽出する能力を持っている。道路クラック画像は「学習したことのない」クラスだが、エッジや輝度変化を検出する低レベル特徴は共通して使える。

# 第2章 レイヤー・構成要素

## 2.1 Conv2D（畳み込み層）

### 畳み込み演算の仕組み

Conv2D はフィルタ（カーネル）と呼ばれる小さな重み行列を画像上でスライドさせ、各位置で内積を計算する。

**具体例（5×5 の入力画像に 3×3 のフィルタを適用）：**

```
入力画像（抜粋）:
1  2  3  0  1
0  1  2  3  0
1  0  1  2  3
0  1  0  1  2
1  2  1  0  1

フィルタ（3×3）:
1  0 -1
1  0 -1
1  0 -1

左上の位置での計算（内積）:
1×1 + 2×0 + 3×(-1)
+ 0×1 + 1×0 + 2×(-1)
+ 1×1 + 0×0 + 1×(-1)
= 1 + 0 - 3 + 0 + 0 - 2 + 1 + 0 - 1 = -4
```

このフィルタは「縦方向のエッジ」を検出する。左列が白（1）で右列が黒（-1）な場所では大きな値が出る。

フィルタをスライドさせながらすべての位置で計算した結果が**特徴マップ（Feature Map）**。

### 複数フィルタ

Conv2D は複数のフィルタを並行して使う。たとえば `Conv2D(64, 3×3)` は 3×3 フィルタを 64 種類学習する。各フィルタは異なる特徴（横エッジ、縦エッジ、斜めエッジ、特定のテクスチャなど）を検出し、64 枚の特徴マップを出力する。

### padding='same' の意味

フィルタをスライドさせると出力は入力より小さくなる（例：5×5 入力 + 3×3 フィルタ → 3×3 出力）。`padding='same'` を指定すると、入力の周囲に 0 を埋め（ゼロパディング）、出力サイズを入力と同じに保つ。

```
padding='valid'（デフォルト）: 出力サイズ = 入力サイズ - フィルタサイズ + 1
padding='same'              : 出力サイズ = 入力サイズ（ゼロパディングで補填）
```

### チャンネルと入力チャンネルの考慮

RGB 画像は 3 チャンネル（R, G, B）を持つ。Conv2D はチャンネルごとに別々にフィルタを適用して足し合わせる。そのためパラメータ数には入力チャンネル数が掛かる。

### パラメータ数の公式

```
(カーネル縦 × カーネル横 × 入力チャンネル数 + 1) × フィルタ数
```

`+1` はバイアス（各フィルタに 1 つ）。

**本講義の計算例（グレースケール入力、1 チャンネル想定）：**

| 層 | 計算式 | パラメータ数 |
|---|---|---|
| Conv2D(64, 7×7) | (7×7×1 + 1) × 64 | 3,200 |
| Conv2D(128, 3×3) | (3×3×64 + 1) × 128 | 73,856 |
| Conv2D(256, 5×5) | (5×5×128 + 1) × 256 | 819,456 |

**重要な性質：** Conv2D のパラメータ数は入力画像の空間サイズ（縦×横）に依存しない。入力を 256×256 から 512×512 に変えても Conv2D のパラメータ数は変わらない（Flatten 後の Dense 層は変わる）。

---

## 2.2 MaxPooling（最大プーリング層）

### 仕組み

指定したウィンドウサイズの領域から最大値だけを取り出し、特徴マップを縮小する操作。

`MaxPool(2×2)` の例：

```
特徴マップ（4×4）:
1  3  2  4
5  6  1  2
3  2  4  1
1  4  3  6

2×2 領域ごとに最大値を取る:
左上(1,3,5,6) → 6
右上(2,4,1,2) → 4
左下(3,2,1,4) → 4
右下(4,1,3,6) → 6

出力（2×2）:
6  4
4  6
```

空間サイズが 1/2 になる（面積で 1/4）。

### なぜ必要か

- **計算量削減：** 後続層に渡す情報量が減り、計算が速くなる。
- **平行移動不変性（Translation Invariance）：** 特徴がわずかにずれた位置にあっても、MaxPooling の範囲内に収まれば同じ出力が得られる。
- **スケール不変性の補助：** 多少のズームや変形に対してロバストになる。

### プールサイズと空間サイズの変化

本講義の CNN では MaxPool(2,2) が 3 回使われる：

```
256×256 → MaxPool → 128×128
         → MaxPool →  64×64
         → MaxPool →  32×32
```

最終的に 32×32×256 の特徴マップが Flatten に渡される。

---

## 2.3 Flatten（平坦化層）

### 仕組み

多次元テンソルを 1 次元ベクトルに変換する。Conv2D・MaxPooling で空間情報を持ったテンソルを、Dense 層が受け取れる形に変換するために使う。

```
形状 (32, 32, 256) → Flatten → 形状 (262144,)
次元数: 32 × 32 × 256 = 262,144
```

### Flatten が Dense のパラメータ数に与える影響

Flatten の出力サイズが Dense 層の入力サイズになる。`Dense(256)` なら：

```
パラメータ数 = (Flatten出力サイズ + 1) × 256
            = (262,144 + 1) × 256
            = 67,109,120  ← 約 6,700 万！
```

入力画像サイズを 512×512 にすると Flatten 後が 1,048,576 になり、Dense(256) のパラメータは約 2.68 億に膨れ上がる。これが「入力サイズの変更が Dense 層にしか影響しない」理由。

---

## 2.4 Dense（全結合層）

### 仕組み

入力ベクトルのすべての要素を出力ベクトルのすべての要素に接続した層。各接続に重みとバイアスがある。

```
出力ベクトル y[j] = activation(Σᵢ w[i][j] × x[i] + b[j])
```

### 本講義での使われ方

| 配置 | 役割 |
|---|---|
| `Dense(512, ReLU)` | Simple NN の隠れ層。特徴を圧縮・変換する |
| `Dense(256, ELU)` | CNN の隠れ層。Flatten 後の高次元特徴を圧縮 |
| `Dense(2, Softmax)` | 出力層。2 クラスの確率を出力する |

### パラメータ数の公式

```
(入力次元 + 1) × 出力次元
```

`+1` はバイアス（出力ニューロン 1 つにつき 1 個）。

---

# 第3章 活性化関数

活性化関数は「なぜ必要か」を先に理解することが重要。

**活性化関数がなければ：** ニューラルネットワークは何層積み重ねても「行列の掛け算の繰り返し」にすぎず、数学的には 1 層の線形変換と等価になる。非線形な問題（「クラックがある」vs「ない」という複雑な境界）を解けない。

活性化関数は非線形性を加えることで、ニューラルネットワークが複雑なパターンを学習できるようにする。

---

## 3.1 ReLU（Rectified Linear Unit）

### 数式

```
ReLU(x) = max(0, x)
```

グラフ：

```
出力
 |     /
 |    /
 |   /
 |  /
 | /
 |/___________→ 入力
0
```

- x > 0：そのまま通す
- x ≤ 0：ゼロにする

### なぜよく使われるか

1. **計算が単純：** max(0, x) の比較だけ。Sigmoid のような指数関数計算が不要。
2. **勾配消失が起きにくい：** x > 0 では勾配が常に 1。Sigmoid や Tanh は大きな値で勾配がほぼゼロになる。
3. **スパースな活性化：** 入力の約半分が 0 になるため、ネットワークが効率的になる。

### Dying ReLU 問題

x ≤ 0 の勾配は常にゼロのため、一度ゼロになったニューロンは永遠に更新されず「死んで」しまう可能性がある。これを Dying ReLU 問題と呼ぶ。ELU はこれを解決する（3.2 節参照）。

---

## 3.2 ELU（Exponential Linear Unit）

### 数式

```
ELU(x) = x              (x > 0)
ELU(x) = α(e^x - 1)    (x ≤ 0)  ※ α = 1.0（デフォルト）
```

グラフ：

```
出力
 |     /
 |    /
 |   /
 |  /
 | /
-α~--------/---→ 入力
           0
```

- x > 0：ReLU と同じ（そのまま通す）
- x ≤ 0：指数関数で滑らかに -α に近づく

### ReLU との違い

| 比較項目 | ReLU | ELU |
|---|---|---|
| x ≤ 0 の出力 | 完全にゼロ | 小さい負の値（滑らか） |
| Dying ReLU | 起きうる | 起きにくい（勾配がゼロにならない） |
| 計算コスト | 低い | やや高い（指数関数を含む） |
| 収束速度 | 普通 | やや速い傾向 |

本講義では CNN の全結合層 `Dense(256, elu)` に使用。ReLU と異なり負の出力も持てるため、平均的な活性化値がゼロに近づき学習が安定しやすい。

---

## 3.3 Softmax

### 数式

```
Softmax(z_i) = e^{z_i} / Σ_j e^{z_j}
```

すべての出力の和が 1 になる（確率分布）。

### 具体例（2 クラス分類）

Dense(2) が出力した生の値（ロジット）が `z = [2.1, -0.3]` の場合：

```
e^{2.1} = 8.17
e^{-0.3} = 0.74

Softmax([2.1, -0.3]) = [8.17 / (8.17 + 0.74), 0.74 / (8.17 + 0.74)]
                     = [0.917, 0.083]
```

「クラックあり（クラス 0）の確率 91.7%、損傷なし（クラス 1）の確率 8.3%」という出力になる。

### なぜ出力層に使うか

- **確率として解釈できる：** 全クラスの合計が 1 になる。
- **最も高い確率のクラスを予測として採用：** `argmax([0.917, 0.083]) = 0`（クラックあり）。

---

# 第4章 正則化・学習安定化

## 4.1 Dropout

### 過学習とドロップアウトの関係

ニューラルネットワークは複雑なため、訓練データを「暗記」してしまいやすい（過学習）。これを防ぐ正則化手法の一つが Dropout。

### 仕組み

学習中の各ミニバッチ処理において、指定した確率 p でニューロンをランダムに無効化（出力を強制的に 0 にする）する。

```
Dropout(0.4) の場合：
各ニューロンを 40% の確率でその回だけ無効化する

通常（無効化なし）：ニューロン A, B, C, D, E 全員が協力して予測
Dropout（40%無効化）：ある回は A, C, E だけ、次の回は B, D だけ、...
```

**直感：** 一人に頼り切るのではなく、誰が欠けても機能するチームを作るイメージ。特定のニューロンへの依存（共適応）を防ぐ。

### 学習時と推論時の違い

- **学習時：** p の確率でニューロンを無効化。残ったニューロンの出力は `1/(1-p)` 倍にスケールアップ（期待値を維持するため）。
- **推論時：** すべてのニューロンを使用する。無効化しない。

### Dropout 率の影響

| 率 | 挙動 | 理由 |
|---|---|---|
| 0（なし） | 過学習しやすい | 正則化なし |
| 0.4〜0.5 | 汎化性能が向上 | 適切な正則化 |
| 0.8（過剰） | 未学習（訓練精度も低い） | 情報が伝わらなくなる |

---

## 4.2 Batch Normalization（バッチ正規化、BN）

### 内部共変量シフト（Internal Covariate Shift）問題

学習中に各層の入力の分布が変化し続ける問題。前の層の重みが更新されるたびに、次の層が受け取るデータの平均・分散が変わる。これにより学習率を大きくできず、収束が遅くなる。

### Batch Normalization の仕組み

ミニバッチ内のデータを使って各層の活性化値を正規化（平均 0、分散 1 に近づける）し、その後スケール・シフトパラメータで調整する。

数式：

```
① ミニバッチの平均を計算：μ_B = (1/m) Σ xᵢ
② ミニバッチの分散を計算：σ²_B = (1/m) Σ (xᵢ - μ_B)²
③ 正規化：x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε)
④ スケール・シフト：yᵢ = γ × x̂ᵢ + β
   （γ と β は学習で求まるパラメータ）
```

`ε`（イプシロン）は分母がゼロになるのを防ぐ小さな定数。

`γ` と `β` があることで「完全に正規化されたままにする必要はない」ことをモデルが学習できる。

### 効果

- 学習が安定し、より大きな学習率が使える。
- 収束が速くなる。
- 正則化効果（Dropout と組み合わせることで過学習をより強く抑制）。
- 重みの初期化に対する感度が下がる。

本講義では各 Conv2D の前に BN を配置している。

### 学習時と推論時の違い

- **学習時：** ミニバッチ内の統計量（μ_B, σ²_B）を使う。
- **推論時：** 学習中に蓄積した移動平均（全データの統計量の近似）を使う。

---

## 4.3 Early Stopping（早期停止）

### 過学習と Early Stopping の関係

エポック数を増やすほどモデルは訓練データに「慣れ」すぎて未知データへの性能が下がる（過学習）。しかし最適なエポック数は事前にわからない。

**Early Stopping** は検証損失の推移を監視し、改善が止まった時点で自動的に学習を停止するコールバック（学習中に呼び出される補助処理）。

### 動作の仕組み

```
エポック 1：訓練損失 0.80、検証損失 0.82 → 改善あり（0.82が過去最低）
エポック 2：訓練損失 0.70、検証損失 0.74 → 改善あり
エポック 3：訓練損失 0.60、検証損失 0.70 → 改善あり
エポック 4：訓練損失 0.50、検証損失 0.72 → 悪化（patience カウント 1）
エポック 5：訓練損失 0.40、検証損失 0.73 → 悪化（patience カウント 2）
...
patience=5 に達したら → 学習停止
```

`restore_best_weights=True` にすると、停止時に最も良い検証損失を記録したエポックの重みに戻す。

### 損失曲線のイメージ

```
損失
 |
 |\ 訓練損失（ずっと下がる）
 | \
 |  \______
 |         \______ ← 最適点
 |    検証損失      \____↗  ← 過学習開始
 |                  ↑
 +-----|---------|-----→ エポック
       最適点   Early Stopping で停止
```

---

## 4.4 ReduceLROnPlateau（学習率の自動調整）

### 目的

学習が進むと損失が「横ばい」になる局面がある。これは現在の学習率では損失の谷底に到達できず、周囲を大きく飛び回っている状態（振動）を示していることが多い。

学習率を小さくすることで、谷底付近をより精密に探索できる。

### 動作の仕組み

```
エポック 1〜5：検証損失が改善している
エポック 6：改善なし（patience カウント 1）
エポック 7：改善なし（patience カウント 2）
エポック 8：改善なし（patience カウント 3 → patience=3 に達した）
→ 学習率を factor 倍（例：0.5）に下げる
→ 学習率 0.0001 → 0.00005
```

設定例：

```python
ReduceLROnPlateau(
    monitor='val_loss',
    factor=0.5,      # 現在の学習率に掛ける係数（0.5 なら半分）
    patience=3,      # 何エポック改善しなければ下げるか
    min_lr=1e-7      # 学習率の下限
)
```

### Early Stopping との使い分け

| 手法 | 役割 |
|---|---|
| ReduceLROnPlateau | 学習が停滞したら学習率を下げて探索を精密化 |
| Early Stopping | 検証損失が本格的に悪化し始めたら学習を停止 |

2 つを組み合わせることで、「学習率を下げてもう少し改善を試みつつ、それでもダメなら停止する」という戦略をとれる。

# 第5章 学習・最適化

## 5.1 AdamW オプティマイザ

### 最適化アルゴリズムの歴史的流れ

**SGD（確率的勾配降下法）：**
```
w ← w - lr × ∂L/∂w
```
シンプルだが、すべてのパラメータに同じ学習率を使う。学習の安定性が低い。

**Momentum（モーメンタム付き SGD）：**
過去の勾配の方向を「慣性」として蓄積し、更新をスムーズにする。

```
v ← β × v + (1-β) × ∂L/∂w   （勾配の指数移動平均）
w ← w - lr × v
```

**Adam（Adaptive Moment Estimation）：**
各パラメータに対して個別の適応的な学習率を計算する。

```
m ← β₁ × m + (1-β₁) × ∂L/∂w        （一次モーメント：勾配の指数移動平均）
v ← β₂ × v + (1-β₂) × (∂L/∂w)²     （二次モーメント：勾配の二乗の指数移動平均）

m̂ = m / (1 - β₁ᵗ)   （バイアス補正）
v̂ = v / (1 - β₂ᵗ)   （バイアス補正）

w ← w - lr × m̂ / (√v̂ + ε)
```

- β₁=0.9, β₂=0.999 が一般的なデフォルト値
- 勾配が大きいパラメータの学習率は自動的に小さくなり、小さいパラメータの学習率は大きくなる
- 設定した学習率 lr は「最大学習率の上限」として機能する

**AdamW（Adam + Weight Decay）：**

通常の Adam では L2 正則化（重みの二乗和にペナルティ）をかけると適応的学習率の仕組みと干渉する問題がある。AdamW は重み減衰（Weight Decay）を勾配ではなくパラメータ更新に直接適用することでこれを修正。

```
w ← w - lr × m̂ / (√v̂ + ε) - lr × λ × w
                                    ↑
                             weight decay 項（パラメータそのものに比例）
```

### 本講義での設定

`lr=0.0001`（= 10⁻⁴）が使われている。AdamW の安定性により、比較的小さな学習率でも確実に収束できる。

---

## 5.2 過学習（Overfitting）

### 直感的な説明

試験前に「過去問だけ」を丸暗記した学生をイメージする。過去問では満点を取れるが、少し違う問題が出ると全く解けない。機械学習モデルが訓練データを「暗記」してしまうのがこれに相当する。

### 訓練損失と検証損失の乖離

```
良い学習（汎化している）：
  訓練損失 → 0.3
  検証損失 → 0.35  （近い値）

過学習：
  訓練損失 → 0.1   （とても低い）
  検証損失 → 0.8   （大きく乖離）
```

### 未学習（Underfitting）との違い

```
未学習（Underfitting）：
  訓練損失 → 0.7
  検証損失 → 0.7   （両方高い）
  原因：モデルが単純すぎる・学習不足
```

### 過学習の原因

- モデルのパラメータ数がデータ数に対して多すぎる（容量過剰）
- エポック数が多すぎる
- データ量が少ない

### 対策

| 対策 | 仕組み |
|---|---|
| Dropout | ランダムに一部ニューロンを無効化し、特定パターンへの依存を防ぐ |
| Batch Normalization | 活性化値を正規化し学習を安定化 |
| Early Stopping | 検証損失が悪化したら停止 |
| データ拡張 | 仮想的にデータを増やす |
| Weight Decay（L2 正則化） | 重みが大きくなりすぎることにペナルティ |

---

## 5.3 ハイパーパラメータ（学習率・バッチサイズ・エポック数）

**ハイパーパラメータ**とは、モデルが学習によって決める「パラメータ（重み）」とは異なり、人間が事前に設定する設計上の値。

### 学習率（Learning Rate）

勾配降下法で一度の更新でどれだけ移動するかを制御する最重要ハイパーパラメータ。

```
損失の地形イメージ：
                ___
               /   \
              /     \_______
学習率小：  •→→→→→→→→→→→→→最小値（小さな歩幅で慎重に）
学習率適切：•→→→→最小値
学習率大：  •→←→←→←→←（谷を飛び越えて振動）
```

| 学習率 | 挙動 |
|---|---|
| 大きすぎる (0.1) | 損失が振動・発散 |
| 適切 (0.0001) | 安定して収束 |
| 小さすぎる (0.000001) | 収束が非常に遅い |

### バッチサイズ（Batch Size）

1 回の重み更新に使うサンプル数。本講義では 8。

- **バッチサイズが小さい（例：8）：** 勾配の推定にノイズが多い → 損失の谷底に小さな変動があっても脱出できる → 汎化性能が高い傾向。ただし学習が不安定。
- **バッチサイズが大きい（例：256）：** 勾配の推定が安定し計算効率も高い → シャープな最小値（汎化しにくい）に収束しやすい傾向。

### エポック数（Epochs）

訓練データ全体を何回繰り返して学習するか。

- **Simple NN：200 エポック**  
  全結合モデルは収束が遅いため多くのエポックが必要。
- **CNN・ResNet50：5 エポック**  
  CNN は特徴抽出が効率的なため少ないエポックでも十分な性能を得られることが多い。

---

## 5.4 データ拡張（Data Augmentation）

### 目的

本講義のデータセットはクラックあり 167 枚・損傷なし 224 枚と少ない。データ拡張は既存の画像に変換を加えて仮想的にデータを増やす。

**効果：**
- 過学習を抑制（モデルが特定の画像を「暗記」しにくくなる）
- 汎化性能の向上（様々な条件での画像に対応できるようになる）

### 本講義で使用した変換

| 手法 | 内容 | 道路クラック検出での効果 |
|---|---|---|
| 水平反転 | 左右に鏡像反転 | クラックの向きに依存しなくなる |
| 垂直反転 | 上下に鏡像反転 | 撮影方向への過学習を防ぐ |
| シフト | 縦横にランダムにずらす | 画像内の位置への依存をなくす |
| ズーム | ランダムに拡大・縮小 | スケールへの依存をなくす |
| 回転 | ランダムに角度を変えて回転 | 傾いた画像にも対応できる |

**注意点：** データの意味が変わってしまう変換はしない（例：「上下反転でも意味が変わらない」なら使えるが、文字認識で「p」が「d」に変わるような反転は不適切）。道路画像は向きに対してある程度不変なので、これらの変換はすべて適切。

# 第6章 評価指標・エラー分析

## 6.1 混同行列（Confusion Matrix）と TP/TN/FP/FN

### 2 クラス分類の結果は 4 通り

|  | 予測：Positive（クラックあり） | 予測：Negative（損傷なし） |
|---|---|---|
| **実際：Positive（クラックあり）** | **TP（True Positive：真陽性）** | **FN（False Negative：偽陰性）** |
| **実際：Negative（損傷なし）** | **FP（False Positive：偽陽性）** | **TN（True Negative：真陰性）** |

- **TP（真陽性）：** 本当にクラックがある → 正しく「クラックあり」と予測 ✅
- **TN（真陰性）：** 実際に損傷なし → 正しく「損傷なし」と予測 ✅
- **FP（偽陽性）：** 実際は損傷なし → 誤って「クラックあり」と予測 ❌（誤検知）
- **FN（偽陰性）：** 本当はクラックあり → 誤って「損傷なし」と予測 ❌（見逃し）

### 道路管理における各エラーのコスト

- **FP（誤検知）：** 正常な道路を「損傷あり」と言ってしまう → 無駄な修繕費用が発生するが人命への影響は少ない
- **FN（見逃し）：** 危険なクラックを「問題なし」と言ってしまう → 事故・陥没などの深刻な被害につながる

**→ FN のコストが圧倒的に高いため、Recall（見逃しを減らす指標）が最重要。**

### Error Analysis セクション

本講義では TP・TN・FP・FN それぞれに分類された画像を実際に表示する。誤分類された画像を目で見ることで「なぜ間違えたか」のパターン把握ができる。例えば「照明が暗い画像を誤検知しやすい」など、改善の手がかりが得られる。

---

## 6.2 Precision（適合率）

### 計算式

```
Precision = TP / (TP + FP)
```

### 意味

「損傷ありと予測した中で、実際に損傷ありだった割合」。

**具体例：** 100 か所に「クラックあり」と予測したうち、実際にクラックがあったのは 70 か所 → Precision = 0.70

Precision が低い = 誤検知（FP）が多い = 「大丈夫な道路を何度も損傷ありと言ってしまう」状態。

---

## 6.3 Recall（再現率）

### 計算式

```
Recall = TP / (TP + FN)
```

### 意味

「実際に損傷ありの箇所のうち、正しく損傷ありと検出できた割合」。

**具体例：** 実際にクラックがある 50 か所のうち、45 か所を正しく検出できた → Recall = 0.90

Recall が低い = 見逃し（FN）が多い = 「危険なクラックを見落としてしまう」状態。

### クラス不均衡下での Recall = 0 の問題

本講義の ResNet50 実験で Recall = 0 が観察された。これは「すべての画像を損傷なしと予測する」という最悪の状態。

```
全 100 件を「損傷なし」と予測した場合（クラックあり 30 件・損傷なし 70 件）：
TP = 0, TN = 70, FP = 0, FN = 30

Accuracy = (TP + TN) / 全体 = 70/100 = 70%  ← 高く見える
Recall = 0 / (0 + 30) = 0%  ← 完全に使えない
```

Accuracy だけ見ると「70% の精度」に見えるが、実際には一件もクラックを検出できていない。

---

## 6.4 F1 スコア

### 計算式

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

これは Precision と Recall の**調和平均**。

### なぜ算術平均ではなく調和平均を使うか

算術平均では「片方が極端に高くても大きく見える」問題がある。調和平均は両方が高くないと高い値にならない。

**例：**

```
ケース A：Precision = 0.9, Recall = 0.1
  算術平均：(0.9 + 0.1) / 2 = 0.5  ← 良さそうに見える
  F1（調和平均）：2 × 0.9 × 0.1 / (0.9 + 0.1) = 0.18  ← 低い（正しく評価）

ケース B：Precision = 0.8, Recall = 0.7
  F1：2 × 0.8 × 0.7 / (0.8 + 0.7) = 0.747
```

### クラス不均衡データでの意義

全部「損傷なし」と予測した場合（損傷あり 30%、損傷なし 70%）：

```
Accuracy = 70%   ← 高く見える
F1（損傷ありクラス） = 0   ← 使えないことが明確
```

不均衡データでは F1 スコアが Accuracy より信頼できる評価指標。

## 6.5 クラス不均衡（Class Imbalance）

### 定義

データセット内のクラス間でサンプル数に大きな偏りがある状態。

### 本講義での状況

| クラス | 枚数 |
|---|---|
| 000_damage（クラックあり） | 167 枚 |
| 001_no_damage（損傷なし） | 224 枚 |

このデータセット自体は 167:224 ≈ 43%:57% でまだ比較的均衡。しかし**現実の道路では損傷箇所は全体の 1〜5% 程度**と非常に少ない。

### 見かけ上の高精度の問題（問題 1 の核心）

損傷 1%・正常 99% の現実的なデータで、すべて「正常」と予測するダミーモデルを考える：

```
Accuracy = 99%  ← 「精度 99%！」
Recall（損傷クラス） = 0%  ← 完全に無意味
```

「精度 99% の道路点検ソフト」の主張を鵜呑みにしてはいけない理由がこれ。

### 対策（本講義では言及）

- Recall や F1 スコアを評価指標として使う。
- クラスに重み付け（Class Weight）を設けて、少数クラスの誤分類ペナルティを大きくする。
- オーバーサンプリング（少数クラスのデータを増やす）やアンダーサンプリング（多数クラスのデータを減らす）。

## 7.1 データ準備（Data Preparation）

### データセットの構成

本講義のデータは Google Drive 上の `lecture_data.tar` で提供される。

```
lecture_data/
```

### 前処理の流れ

2. **画像の読み込み：** PIL または OpenCV で画像ファイルを読み込む。

4. **正規化：** ピクセル値を [0, 255] から [0.0, 1.0] に変換する（`/ 255.0`）。

5. **NumPy 配列化：** `(枚数, 縦, 横, チャンネル)` の 4 次元配列に変換する。
6. **ラベル作成：** 各画像に対して正解ラベル（0 または 1）を割り当てる。
7. **データ分割：** scikit-learn の `train_test_split` で訓練・検証・テストに分割する。

## 7.2 画像操作（NumPy による配列操作・反転・クロップ）

### コンピュータが画像を「見る」方法

コンピュータにとって画像は数値の 3 次元配列（テンソル）。RGB 画像は次の形状を持つ：

```
shape: (高さ, 幅, チャンネル) = (256, 256, 3)

img[y][x][0] = 赤成分（0〜255）
img[y][x][1] = 緑成分（0〜255）
img[y][x][2] = 青成分（0〜255）
```

### NumPy による操作

NumPy の配列スライシングを使うことで、画像の変換を数行で書ける。

**水平反転（左右反転）：**
```python
flipped = np.fliplr(img)
# または
flipped = img[:, ::-1, :]  # 幅方向を逆順に
```

**垂直反転（上下反転）：**
```python
flipped = np.flipud(img)
# または
flipped = img[::-1, :, :]  # 高さ方向を逆順に
```

**ランダムクロップ（切り抜き）：**
```python
h, w = img.shape[:2]
crop_h, crop_w = 200, 200  # 切り抜きサイズ
y = random.randint(0, h - crop_h)
x = random.randint(0, w - crop_w)
cropped = img[y:y+crop_h, x:x+crop_w, :]
```

### なぜ NumPy を使うか

ピクセル操作を Python の `for` ループで書くと数秒〜数分かかる。NumPy はメモリ上で C 言語レベルの高速演算を行うため、同じ処理がミリ秒単位で完了する。

## 7.3 ImageNet（事前学習データセット）

### 概要

ImageNet は Stanford University が作成した大規模画像データセット。

| 項目 | 数値 |
|---|---|
| クラス数 | 1,000 クラス（犬・猫・車・建物・植物など） |
| 学習データ | 約 120 万枚 |
| 検証データ | 約 5 万枚 |
| テストデータ | 約 10 万枚 |
| 画像サイズ | 様々（224×224 に統一されることが多い） |

### ImageNet Large Scale Visual Recognition Challenge（ILSVRC）

2010〜2017 年に毎年開催された画像認識コンテスト。2012 年に AlexNet が CNN を用いてエラー率を大幅に下げ、深層学習ブームの火付け役となった。ResNet は 2015 年の優勝モデル。

### 転移学習における意義

ImageNet で学習されたモデルは低レベルな特徴（エッジ、輝度、色のグラデーション）から高レベルな特徴（テクスチャ、形状、物体の一部）まで汎用的な特徴抽出器になっている。

これを道路クラック検出に転用すると：
- 「エッジを検出する」という能力は、クラックの検出にそのまま使える。
- 少量のデータでも、この「共通の基礎能力」を活かして高精度を達成できる。

## 8.2 NumPy

### なぜ Python の標準リストではなく NumPy を使うか

Python のリストは汎用的（要素が異なる型でも可）だが遅い。NumPy の ndarray は**同一型の要素の固定サイズ配列**で、メモリが連続して確保され C 言語の配列と同等の速度で演算できる。

```
1000 万要素の配列の足し算：
Python リスト：約 1 秒
NumPy ndarray：約 0.01 秒（100 倍以上速い）
```

### 主な特徴

```
GPU：低性能だが数千コア → 単純な演算を大量並列処理（行列計算に最適）
```
